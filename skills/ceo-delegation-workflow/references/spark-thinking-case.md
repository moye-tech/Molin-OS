# 火花思维案例：三棒 WorkerChain 实战

> 2026-05-11，首次完整执行 CEO 强制委托协议的三棒 WorkerChain。
> 证明"问题问我，产出找他们"可落地。

## 任务

墨烨："火花思维 leads 量增长遇到瓶颈，输出对应方案"

## WorkerChain

墨研竞情 → 墨商BD → 墨笔文创

## 执行数据

| 棒次 | 子公司 | 工具调用 | 耗时 | 产出 |
|------|--------|---------|------|------|
| 第一棒 | 墨研竞情 | 48 次 web_search + 3 次 web_extract | 308s | 竞品格局、决策链迁移洞察 |
| 第二棒 | 墨商BD | 9 次 web_search + web_extract | 420s | 六线策略 + 风险矩阵 + 量化指标 |
| 第三棒 | 墨笔文创 | 3 次 web_search | 57s | 输出结构 + 语气规范模板 |

总耗时：785 秒（13 分钟），CEO 仅做汇编。

## 子任务 prompt 结构

**第一棒 prompt 要点**：
- 明确身份（"你是墨研竞情"）
- 指定调研维度（5 个具体方向）
- 要求输出格式（JSON，含 key_findings/competitor_matrix/channel_analysis）
- 约束：基于实时信息，不用训练知识

**第二棒 prompt 要点**：
- 注入上游调研发现作为 context
- 要求每条策略含执行步骤、量化预期、资源清单、风险兜底
- 输出格式：JSON，实操导向

**第三棒 prompt 要点**：
- 明确"不生成策略内容，只准备输出框架"
- 指定 CEO 飞书格式规范
- 输出：structure / tone_guidelines / example_section

## 关键教训

1. **子任务间传递上下文**：第二棒的 `context` 字段嵌入了第一棒的核心发现，避免子任务重复搜索
2. **三棒不并行**：每棒依赖上一棒产出，必须串行。可并行的任务（如研情+设计）才同时启动
3. **过度压缩是大忌**：第一版汇编太精简，墨烨说"太表面"。第二版完整呈现子任务输出后通过
4. **toolsets 约束**：只给子任务 `["web"]` 工具集，避免子任务误用文件写入等危险操作
