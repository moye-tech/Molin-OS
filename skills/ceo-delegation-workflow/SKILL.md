# CEO 委托工作流

## 触发条件

收到任何包含以下动词的用户消息时触发：
写 / 做 / 生成 / 创作 / 设计 / 调研 / 分析 / 开发 / 配音 / 上架 / 记录 / 发布

## 强制决策树

Step 1 · 分类
  A. 闲聊/情绪/策略讨论 → 直接对话
  B. 含产出动词 → 必须委托

Step 2 · 查历史
  有成功先例 → 复用上次Worker组合
  无先例 → 查 WorkerChain 标准组合表

Step 3 · 规划 WorkerChain
  单一领域 → 单Worker
  跨领域 → WorkerChain 串联

Step 4 · 委托执行
  使用 delegate_task 工具，每个子公司一个子任务
  第一个子任务完成后再启动后续（串行依赖时）
  可并行的任务同时启动

Step 5 · 协调汇总
  收集所有子任务输出
  CEO 仅做汇编（格式调整、去重、补充过渡）
  不自生成策略/分析/内容

## 3秒自查清单（发送任何回复前）

□ 我是在自己生成内容（文案/代码/分析）吗？→ 是：停止，委托
□ 我用的是训练知识回答实时问题吗？→ 是：停止，先调墨研竞情
□ 这个任务只用了一个Worker但应该用多个吗？→ 查组合表
□ 我告诉创始人谁在做、多久、产出在哪了吗？→ 补充三要素

## WorkerChain 标准组合（高频）

| 任务 | 链路 | 并行 |
|------|------|------|
| 内容营销 | 墨研竞情→墨笔文创→墨图设计 | 研情+设计 |
| 竞品报告 | 墨研竞情→墨测数据→墨笔文创 | 研情+数据 |
| 电商上架 | 墨研竞情→墨笔文创→墨图设计→墨链电商 | 研情+设计 |
| 增长策略 | 墨研竞情→墨商BD→墨笔文创 | 否 |
| 开发部署 | 墨码开发→墨安安全→墨维运维 | 安全+运维 |

## 常见错误

1. CEO 看到"帮我分析"直接用自己的训练知识回答 → 应先调墨研竞情做实时搜索
2. CEO 只用一个 delegate_task 而不是按 WorkerChain 拆成多个子任务
3. CEO 对子任务输出过度压缩/改写 → 汇编即可，不改内容。墨烨说"太表面" = 压缩损失了细节
4. 子任务 toolsets 错误 → delegate_task 必须指定 `toolsets: ["web"]` 限制子任务工具范围
5. 串行依赖的任务并行启动 → 第二棒依赖第一棒产出时必须串行；不依赖的才并行

## delegate_task 工具要点

```python
delegate_task(
    goal="你是墨研竞情。深度调研火花思维竞品格局...",
    context="上游发现:斑马57%份额，决策链已迁移...",  # 注入上游产出
    toolsets=["web"],  # 限制工具范围，防止误操作
)
```

- `goal`: 子任务身份 + 产出要求（不是系统提示）
- `context`: 上游 Worker 的发现，避免子任务重复搜索
- `toolsets`: 必须指定，不给子任务文件写入权限
