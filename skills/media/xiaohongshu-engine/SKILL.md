# xiaohongshu-engine · 小红书内容引擎
# 归属: Agent A (media) 独占 · 版本: v1.0

## 触发词（Agent A收到这些词时必须调用）
触发词: 小红书|xhs|笔记|爆款|收藏|选题|小红书发文
触发词: 平台算法|算法变化|发布时间|标签优化

## 不调用时机
- 抖音视频内容 → 改用 video-script-engine
- 纯竞品分析 → 先调Agent D的research-engine获取数据

## 调用格式
INPUT: { "topic": "主题", "tone": "干货型|故事型|对比型", "target": "受众描述" }
OUTPUT: { "titles": ["×3"], "body": "正文", "tags": ["×5"], "cover_prompt": "封面提示词", "best_time": "发布建议" }

## 执行流程
Step 1: 调用web-scraper抓同话题近7天TOP10竞品笔记
Step 2: 提取标题公式、高频词、互动数据
Step 3: 生成差异化内容 + 3个备用标题
Step 4: SEO优化（关键词密度、标签配置）
Step 5: 输出结构化JSON

## 当前经验规则（动态更新）
- 含数字的标题收藏率+23%（2026-04统计）
- 晚上20:00-22:00发布互动率最高
- 5个标签比3个标签曝光多约15%
- 开篇前3行必须有钩子（问句/数字/反差）
- 字数800-1200字完读率最高
