# 墨育 · AI 教育引擎

## 概述

墨育是赫墨斯 OS 的教育智能模块。基于前沿 AI 教育研究和开源项目的最佳实践：

- **DeepTutor** (HKUDS, ⭐23.2k) — 多模态 AI 个性化辅导系统，含学习引擎、知识库、TutorBot、测验生成
- **Human Skill Tree** (24kchengYe, ⭐517) — 基于认知科学的终身学习技能树框架

### 核心理念

```
AI 时代的教育 = 个性化路径 × 间隔重复 × 主动回忆 × 知识图谱
                       ↓
             墨育引擎（Hermes + 记忆系统 + 工具）
```

## 核心功能

### 1. 课程设计（课程大纲生成）

```python
# 基于 Bloom 分类法生成结构化课程
# 输入：主题 + 目标受众 + 学习时长
# 输出：模块化课程大纲（含章节、目标、评估方式）
```

**方法论：** Bloom 认知分类（记忆→理解→应用→分析→评价→创造）

| 层次 | 认知目标 | 输出形式 |
|:----:|:---------|:---------|
| 📖 记忆 | 回忆事实、术语、概念 | 术语表、知识点列表 |
| 📗 理解 | 解释、概括、对比 | 总结文、对比表 |
| 🛠️ 应用 | 在新场景中运用 | 案例分析、实操练习 |
| 🔍 分析 | 分解、归因、区分 | 结构图、因果链 |
| ⚖️ 评价 | 评判、辩护、推荐 | 评估报告、方案评选 |
| 🎨 创造 | 组合、设计、构建 | 方案设计、产品原型 |

### 2. AI 辅导（个性化教学）

基于 DeepTutor 的多模式教学法：

| 模式 | 适用场景 | 方法 |
|:----:|:---------|:-----|
| 💬 Chat | 概念讲解、答疑 | Socratic 问答法，不直接给答案 |
| 🧠 Deep Solve | 复杂问题推理 | 分解→推理→验证，结构化输出 |
| 📝 Quiz | 知识检验 | 自动生成选择题/填空题/简答题 |
| 🔬 Research | 深度研究 | 多轮检索+综合报告 |
| 📊 Visualize | 可视化理解 | 图表/思维导图/概念图 |

### 3. 学习路径规划

**Skill Tree 方法（来自 human-skill-tree 框架）：**

```yaml
学习路径:
  总目标: "掌握 Python 数据分析"
  时间: 12 周
  模块:
    - 阶段1: Python 基础 (2周)
      前置知识: 无
      里程碑: 能写50行脚本
    - 阶段2: Pandas + Numpy (3周)
      前置知识: 阶段1
      里程碑: 能处理真实CSV数据
    - 阶段3: 数据可视化 (2周)
      前置知识: 阶段2
      里程碑: 能产出可视化报告
    - 阶段4: SQL + 数据库 (3周)
      前置知识: 阶段1
      里程碑: 能独立完成数据分析项目
    - 阶段5: 实战项目 (2周)
      前置知识: 全部
      里程碑: 完成端到端数据分析项目
```

### 4. 间隔重复（Spaced Repetition）

基于认知科学最有效的学习方法：

| 间隔 | 复习内容 |
|:----:|:---------|
| 1 天 | 初次学习内容 |
| 3 天 | 修正后的理解 |
| 7 天 | 应用场景 |
| 14 天 | 与已有知识连接 |
| 30 天 | 综合回顾 |

生成 Anki 兼容的问答卡格式：
```markdown
Q: 什么是间隔重复？
A: 间隔重复是一种基于记忆曲线安排复习时间的技巧，
   在记忆即将消退时进行主动回忆，以最大化记忆效率。
```

### 5. 互动测验生成

```markdown

## 选择题

A. tuple  B. str  C. list  D. int

## 填空题

          tuple 是不可变的（创建后不能修改）。
```

## 教学方法论（学术基础）

以下原则来自 DeepTutor 引用的 2025 PNAS 研究和 human-skill-tree 引用的元分析：

1. **Socratic 引导** — 不给直接答案，通过追问引导学生自己发现
2. **主动回忆** — 让学生主动提取记忆，而非被动重读
3. **间隔重复** — 按遗忘曲线安排复习
4. **细粒度反馈** — 不只说"对/错"，而是指出具体错误位置和原因
5. **脚手架式教学** — 从简单到复杂，逐步撤除辅助
6. **多模式学习** — 文本+图表+代码+交互，不同模式强化的不同认知路径

## 常见陷阱

1. **忽略前端评估** — 不先了解学生现有水平就设计课程是无效的
2. **一次性太多** — 每节课的知识点不超过 5-7 个（认知负荷理论）
3. **没有实践环节** — 只讲不练，学习效果大打折扣
4. **跳过复习计划** — 没有间隔重复的学习，1 周后遗忘 70%
5. **一刀切的教学方法** — 不同学生需要不同的教学法（视觉型/听觉型/动觉型）
