# tech-delivery · 技术交付引擎
# 归属: Agent C (side) 独占 · 版本: v1.0

## 技能身份
副业技术方案的技术评估和交付执行引擎。
确定项目技术路线、工作量评估和交付标准。

## 触发词
触发词: 技术方案|实现方案|代码|开发|怎么做|技术路线
触发词: 可以做什么|技术评估|工作评估|开发周期

## 服务能力矩阵
| 方向 | 范围 | 典型周期 |
|------|------|---------|
| Prompt工程 | 提示词设计/优化/测试（DeepSeek/Qwen/GPT） | 1-3天 |
| RAG系统 | 知识库搭建（文档解析/向量化/检索优化） | 5-14天 |
| Agent开发 | 基于Hermes/AutoGen/LangGraph的Agent定制 | 3-7天 |
| 自动化脚本 | Python/n8n自动化流程 | 2-7天 |
| 内容批量 | 基于LLM的批量内容生产脚本 | 1-3天 |

## 技术选型原则
优先: 国内模型（DashScope/DeepSeek）+ 自托管方案（合规）
其次: 国际API（需确认客户不在数据敏感行业）
禁止: 向客户推荐需要特殊网络的工具（不稳定，售后难）

## 交付标准包
1. 主交付物（代码/文档/系统）
2. 使用说明书（中文，无技术术语）
3. 测试记录（已验证场景清单）
4. 7天问题响应承诺
5. 交付清单确认单

## 输出格式
```json
{
  "feasibility": "可行|需调整|不可行",
  "tech_stack": ["推荐技术栈"],
  "effort_days": 5,
  "deliverables": ["交付物清单"],
  "risks": ["技术风险"]
}
```

## 当前经验规则（动态更新）
- 含数据库设计的项目比纯脚本项目多预留40%时间
- 第一版配置期+对接客户期占用总工期30%
