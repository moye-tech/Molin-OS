    18|# Changelog Automation
    19|
    20|Patterns and tools for automating changelog generation, release notes, and version management following industry standards.
    21|
    22|## Use this skill when
    23|
    24|- Setting up automated changelog generation
    25|- Implementing conventional commits
    26|- Creating release note workflows
    27|- Standardizing commit message formats
    28|- Managing semantic versioning
    29|
    30|## Do not use this skill when
    31|
    32|- The project has no release process or versioning
    33|- You only need a one-time manual release note
    34|- Commit history is unavailable or unreliable
    35|
    36|## Instructions
    37|
    38|- Select a changelog format and versioning strategy.
    39|- Enforce commit conventions or labeling rules.
    40|- Configure tooling to generate and publish notes.
    41|- Review output for accuracy, completeness, and wording.

    43|
    44|## Safety
