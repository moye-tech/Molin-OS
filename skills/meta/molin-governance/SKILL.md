# 墨麟治理规则

## 四级审批

| 级别 | 触发 | 行为 | 示例 |
|------|------|------|------|
| **L0 绿灯** | 低风险·低金额 | 自动执行，事后记录 | 回复闲鱼消息、发布定时内容、数据查询 |
| **L1 提示** | 中等风险 | 执行后通知老板 | 新服务上架、定价 ¥100-500、修改商品描述 |
| **L2 确认** | 高风险 | 执行前需老板确认 | 报价 >¥500、投标猪八戒、对客户承诺交付日期 |
| **L3 董事会** | 战略级 | 讨论后决策 | 新业务线、月预算调整 >30%、不再接某类单 |

## 预算控制

```
月度总预算：¥1,360
  ├── 内容中心  ¥250  (小红书/视频/SEO/AI绘画/PPT)
  ├── 商业中心  ¥280  (PRD/BP/市场/财务/战略)
  ├── 增长中心  ¥160  (闲鱼/销售/营销/BD)
  ├── 工程中心  ¥480  (开发/测试/审查)
  ├── 情报中心  ¥160  (搜索/OSINT/监控/趋势)
  └── 运营中心  ¥30   (DevOps/记忆)

规则：
- 70% 消耗时 → 飞书通知老板
- 90% 消耗时 → 该部门自动暂停，需老板手动恢复
- 每月1日重置
```

## 决策审计

所有 L1+ 决策记录格式：
```
[时间] [级别] [部门] [操作] [原因] [结果]
```

示例：
```
2026-05-04 14:30 L1 增长中心 新商品上架"AI绘画-赛博朋克风格" 市场热度高 待客户反馈
```

## 回滚机制

误操作可回滚的场景：
- 误发内容 → 撤回/删除
- 误报价 → 修正并通知客户
- 误上架商品 → 下架
- 配置变更 → 从备份恢复
