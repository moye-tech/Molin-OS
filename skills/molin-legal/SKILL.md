# 墨律法务 · AI 法律助手

## 概述

墨律法务是赫墨斯 OS 的法律智能模块，基于 ai-legal-claude ⭐1.2k 项目的 13 个专业技能适配而来。覆盖从合同审查到合规审计的全链条法务需求。

### 核心能力

| 模块 | 功能 | 适用场景 |
|------|------|---------|
| **合同审查** | 5并行子代理分析+综合报告 | 收到任何合同时首选 |
| **风险评估** | 逐条款1-10评分+财务敞口估算 | 重要合同深度审查 |
| **NDA生成** | 定制化保密协议(4种类型) | 合作前需要NDA |
| **隐私政策** | 网站扫描+GDPR/CCPA合规生成 | 产品或SaaS上线前 |
| **合规审计** | 7大法规框架审查 | GDPR/CCPA/ADA等检查 |
| **TOS生成** | 16节服务条款生成 | 网站/SaaS发布 |
| **合同对比** | 新旧版本逐行对比 | 合同修订后检查 |
| **条款简化** | Legalese→通俗语言 | 非法律人员理解合同 |
| **谈判策略** | 替换语言+话术+让步策略 | 谈判前准备 |
| **缺失条款** | 按合同类型检查缺失保护 | 收到对方模板合同 |
| **自由职业** | 14维度审查+公平评分 | 自由职业者接单前 |
| **商业协议生成** | 10种商业协议模板 | 需要起草合同时 |
| **PDF报告** | 专业报告(含评分仪表盘) | 需要输出给客户/律师 |

## 工作流程

### 合同审查全流程

```
用户提供合同 → 文件读取 → 分类合同类型 → 
  ┌─ 子代理1: 条款分析(20%)
  ├─ 子代理2: 风险评估(25%)  ← 最高权重
  ├─ 子代理3: 合规检查(20%)
  ├─ 子代理4: 义务映射(15%)
  └─ 子代理5: 建议生成(20%)
  └→ 聚合 → 统一报告
```

### 风险评分体系

所有风险采用 **1-10 评分 + R/Y/G 色标**：

| 等级 | 评分 | 含义 | 行动 |
|:----:|:----:|------|:----:|
| 🔴 严重 | 8-10 | 签约风险/损失重大 | 标记为 MUST CHANGE |
| 🟡 高 | 5-7 | 需谈判改善 | 标记为 SHOULD CHANGE |
| 🟢 中 | 3-4 | 注意但可接受 | 标记为 NICE TO CHANGE |
| ✅ 低 | 1-2 | 标准条款 | 确认无误即可 |

### 合同安全评分

最终输出 0-100 的合同安全分：

| 等级 | 分数段 | 建议 |
|:----:|:------:|:----:|
| A+ | 93-100 | 签署 |
| A | 85-92 | 签署 |
| B+ | 77-84 | 签署(小修订) |
| B | 69-76 | 谈判 |
| C+ | 60-68 | 谈判(重点条款) |
| C | 50-59 | 升级 |
| D | 30-49 | 不签署 |
| F | 0-29 | 拒绝 |

## 关键参考：风险类别（11种）

| 风险名称 | 说明 | 权重 |
|:--------:|:----:|:----:|
| 财务敞口 | 未封顶或过高的财务义务 | 高 |
| 责任转移 | 不当责任转移给己方 | 高 |
| 限制性约定 | 竞业/排他/非招揽过度 | 高 |
| 模糊条款 | 定义不清晰、范围不定 | 中 |
| 缺失保护 | 应有条款缺失 | 高 |
| 单方条款 | 权利义务不对等 | 中 |
| 无上限责任 | 未设赔偿责任上限 | 高 |
| 宽泛赔偿 | 赔偿范围无合理限制 | 高 |
| 自动续约 | 自动续约+长通知期 | 中 |
| 竞业过度 | 竞业范围/时间/地域不合理 | 中 |
| 定义地雷 | 关键定义陷阱 | 高 |

## 常见陷阱

1. **未先读取文件就分析** — 必须先用 `read_file` 获取合同内容
2. **跳过合同类型识别** — 不同合同类型(SaaS/雇佣/MSA/NDA/合伙/租赁/供应商)的检查清单不同
3. **忽略管辖法律** — 不同司法管辖区(中国/美国/EU)的法律要求不同，需明确
4. **缺少法务免责声明** — 每次输出必须附带"AI辅助，非法律专业意见"声明
5. **直接输出评分而不给行动建议** — 评分只是开始，必须有可执行的下一步
