# Molin-OS 操作系统（v3.0 · 扁平化）

> 版本: 3.0 · 更新: 2026-05-17
> 原则: 一个系统，一页可读。子目录全部消灭，文件名区分业务线。

---

## 一、执行模型

```
Hermes（大脑）→ skill（专业知识）→ terminal（神经）→ Obsidian 产出/ 输出
```

所有 Agent 输出写入 `产出/业务线｜主题.md`（平坦结构，无子目录）。
输出规范详细见 `产出写入规范.md`（v3.0）。

## 二、全局规则

- **8 根目录 · 零子目录**: `决策` `知识` `流程` `成果` `报告` `配置` `产出` `学习档案`
- 文件命名: `业务线｜具体内容.md`（全角竖线，不重复目录名）
- 日报: 滚动更新单文件（`银月｜日报.md`），不按日期新建
- Gatekeeper 终检是对外输出的最后一道防线
- scheduler.yaml 是唯一调度源

## 六、Vault 目录结构

```
Vault/
├── 决策/       ← 不可逆选择: 架构、战略、技术选型
├── 知识/       ← 研究分析、领域知识、学习笔记
├── 流程/       ← SOP、系统集成、管道文档
├── 成果/       ← 周报、里程碑
├── 报告/       ← KPI看板、定期报告
├── 配置/       ← 系统架构元数据（脚本自动采集）
├── 产出/       ← 所有Agent执行输出（平坦）
└── 学习档案/   ← GitHub精读、论文笔记（平坦）
```

零子目录。命名: `业务线｜具体内容.md`。

输出规范详细见 `产出写入规范.md`（v3.0）。
