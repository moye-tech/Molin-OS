## 一、核心定位

**墨麟 Hermes OS 不是一个独立部署的服务。**  
它是 **Hermes Agent 之上的增强层**——一组定义文件（skills、config、molin 模块、docs）的集合，所有 AI 能力复用 Hermes Agent 自身。

```
┌─────────────────────────────────────────────────────┐
│                    Hermes Agent                      │
│            (AI Agent Orchestration Engine)            │
│        DeepSeek · OpenRouter · 当前模型配置            │
├─────────────────────────────────────────────────────┤
│          墨麟 Hermes OS (增强层 / 定义集合)             │
│                                                       │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐  │
│  │  skills/     │  │  molin/     │  │  config/     │  │
│  │  235个技能模块│  │  Python业务 │  │  公司/渠道/   │  │
│  │             │  │  逻辑层     │  │  治理配置     │  │
│  └─────────────┘  └─────────────┘  └──────────────┘  │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐  │
│  │  docs/       │  │  cron/      │  │  tools/      │  │
│  │  知识文档    │  │  定时任务   │  │  辅助工具     │  │
│  └─────────────┘  └─────────────┘  └──────────────┘  │
└─────────────────────────────────────────────────────┘
```

**关键原则:**
- ❌ 不需要独立 venv / 独立部署
- ❌ 不需要额外 API Key（复用 Hermes 的 DeepSeek 配置）
- ✅ `molin` Python 包已安装到 Hermes 的 venv 中
- ✅ `skills/` 作为 Hermes 技能体系的一部分
- ✅ `config/` 提供定义供 AI 理解"公司"的结构

## 九、cli.py (molin 命令) 使用说明

> ⚠️ **重要:** `molin` CLI 命令的 handler_map 目前是**占位路由**，大部分命令返回的是"等待 Hermes Agent 调度"的消息。实际执行需要通过 Hermes Agent 的能力（即与我对话）来完成。

**目前可用的模式：**

| 命令 | 实际作用 |
|------|----------|
| `python -c "from molin.core.engine import engine; engine.health_check()"` | ✅ 直接可调用，返回健康状态 |
| `molin ceo strategy` | 🔄 展示战略决策框架（需 Hermes 实际驱动） |
| `molin serve` | 🔄 启动 Web 仪表盘（Flask，端口 8080） |
