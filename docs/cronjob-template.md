# 子公司 cronjob 模板规范

> 所有子公司的定时作业遵循统一结构，类似 Python 中的基类继承模式。
> 新子公司接入时，按此模板创建 cronjob。

---

## 模板结构

每个 cronjob 包含 5 个标准部分：

```
┌─ 1. 角色定义 ─────────────────────────────┐
│ 你是谁、你的职责、工作原则、边界           │
├─ 2. 数据获取 ──────────────────────────────┤
│ 从哪里获取输入（session_search/外部源）    │
├─ 3. 核心逻辑 ──────────────────────────────┤
│ 做什么、怎么做                             │
├─ 4. 输出存储 ──────────────────────────────┤
│ 产出存到 memory，为下游准备接力数据        │
├─ 5. 报告格式 ──────────────────────────────┤
│ 结构化日报卡片                             │
└─────────────────────────────────────────────┘
```

## 创建新子公司 cronjob 的步骤

### Step 1: 定义角色
```markdown
## 背景
你是[墨XX]的[角色名] AI。
[一句话职责描述]
与上下游子公司的协作关系：[上游→你→下游]
```

### Step 2: 确定关联技能
列出该子公司可用的 Hermes SKILL.md。

### Step 3: 确定数据流
- **输入**：session_search 检索上游产出的 memory 记录
- **核心处理**：LLM 分析 + web_search 补充
- **输出**：memory 存储 + 结构化报告

### Step 4: 编写 prompt
使用 cronjob(action='create') 创建。

### Step 5: 注册到 cron/jobs.yaml
```yaml
- name: "[墨XX] [作业名]"
  schedule: "[cron表达式]"
  skills: [...]
  deliver: origin
  repeat: forever
```
