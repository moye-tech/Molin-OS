# Agent SOP 体系搭建记录

## 概述

基于"把每一个 Agent 升级成可持续经营的 AI 业务部门"的目标，建立了标准化四层架构。

## 核心框架

```
Agent
 ├── SOP Layer    —— 标准作业程序
 ├── Cron Layer   —— 自动循环周期
 ├── KPI Layer    —— 结果监控指标
 └── Feedback Layer —— 反思优化闭环
```

## 已创建的技能

### 1. agent-sop-template（元技能）
- 定义 Agent 四层架构模板
- 六件套 SOP 标准（Lead / Execution / QA / Escalation / Growth / Crisis）
- 目录约定与使用方式

### 2. content-sop-pack（内容 Agent 技能包）
- 7 步内容生产流水线（选题 → 钩子 → 结构 → 正文 → 自检 → 适配 → 输出）
- 平台适配矩阵（小红书/公众号/抖音/SEO 不同改写方向）
- QA 评分矩阵（7 维度权重评分）
- Escalation 升级表（6 种场景 + 治理级别）
- Cron 经营节奏（日节奏 + 周节奏）

### 3. gatekeeper-sop（全流量合规门禁）
- 通用质检检查矩阵（6 维度 + 5 项否决条件）
- 平台合规规则（小红书/抖音/公众号/LINE）
- 阻断决策树（≥85 放行 / 70-84 修正 / 60-69 人工 / <60 拦截）
- 拦截记录格式

## 使用方式

创建新 Agent 时：
1. `skill_view('agent-sop-template')` 获取模板
2. 确定需要的 SOP 类型
3. 创建对应的 SOP skill
4. 配置 Cron 经营节奏

## 完整技能体系

```
agent-sop-template/          ← 元模板（所有 Agent 的起手式）
content-sop-pack/            ← 内容 Agent 三件套
gatekeeper-sop/              ← 全流量门禁（跨 Agent 共享）
└── 未来：各 Agent 的专属 SOP 放在对应分类下
```
