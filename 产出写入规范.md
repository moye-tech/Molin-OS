# Molin-OS 产出写入规范（v3.0 · 扁平化）

> 每个 Agent 执行完毕后，必须按本规范写入输出。
> 版本: 3.0 · 更新: 2026-05-17
> **本次升级**: 消灭全部子目录，8个根目录平坦结构，`业务线｜具体内容.md` 命名

---

## 一、Vault 目录结构（分类规则）

```
Vault/
├── 决策/       ← 不可逆的选择（技术选型、架构定稿、战略方向）
├── 知识/       ← 可沉淀的认知（研究分析、领域知识、方法论文档）
├── 流程/       ← 可执行的步骤（SOP、配置、操作手册、系统集成）
├── 成果/       ← 周报/里程碑
├── 报告/       ← KPI看板、定期报告
├── 配置/       ← 系统架构元数据（自动采集，collect_architecture.py）
├── 产出/       ← 【核心】所有 Agent 的执行输出（平坦，无子目录）
└── 学习档案/   ← GitHub精读笔记、论文笔记、学习方法论（平坦，无子目录）
```

**零子目录。** `产出/content/` `产出/edu/` `学习档案/media/github-absorb/` `报告/KPI看板/` 全部消灭。文件名区分业务线。

---

## 二、文件命名规范

```
业务线｜具体内容.md
```

- 分隔符: 全角竖线 `｜`
- 根目录 = 类型，**文件名不再重复体现类型**
- 业务线: `系统` `元瑶`(edu) `银月`(media) `梅凝`(global) `宋玉`(side) `玄骨`(shared/finance) `KPI`

| ✅ 正确 | ❌ 错误 |
|---------|---------|
| `配置/系统｜架构` | `配置/系统｜架构｜配置`（冗余） |
| `报告/KPI｜周报·0511-0516` | `报告/KPI看板/周报`（子目录） |
| `产出/元瑶｜AB测试方案` | `产出/edu/ab_test_plan`（旧子目录） |
| `产出/银月｜日报`（滚动更新） | `产出/2026-05-17.md`（日期命名） |
| `学习档案/元瑶｜教育研究吸收·0517` | `学习档案/edu/github-absorb/deep-note`（旧子目录） |

### 定时写入路径对照表

| 脚本 | 写入目标 | 用途 |
|------|---------|------|
| `sync_memory.py` | `{分类}/` | 对话记忆按MECE分类写入 |
| `obsidian_sync.py` | `报告/` + `{分类}/` | 聚合日报 + Molin输出 |
| `collect_architecture.py` | `配置/` | 系统架构 + Agent配置 + 记忆映射 |
| `relay_to_obsidian.py` | `知识/` | 无Agent cron脚本 relay → 知识库 |

## 四、结构化输出模板（必用）

每个 Agent 输出必须使用以下标准模板：

```markdown
---
agent: {业务线}
type: {输出类型: 日报/方案/分析/报告}
created: {YYYY-MM-DD}
updated: {YYYY-MM-DD}
tags: [{标签1}, {标签2}]
---

# {标题}

## 任务目标
一句话说明这次执行在解决什么问题

## 核心结果
关键数据汇总（数字、指标、状态）

## 详细分析
深度分析内容（可选：拆解数据、洞察发现）

## 执行动作
已执行的操作清单

## 风险与异常
遇到的问题、风险、异常数据

## 可复用知识
下次可复用的经验、洞察、模式
```

## 五、禁止规则

| ❌ 禁止 | 原因 |
|---------|------|
| 写入 `产出/{agent_id}/` 子目录 | 旧版结构，已消灭 |
| 写入 `学习档案/{agent_id}/github-absorb/` | 旧版子目录，已消灭 |
| 写入 `报告/KPI看板/` | 旧版子目录，已消灭 |
| 写入 `agent-outputs/` | 已废弃的旧路径 |
| 写入 `Agents/` `02_Agent_Outputs/` | 旧版顶级目录，已消灭 |
| 文件名含日期作为主名 | 只有一次性报告可含日期后缀，日报必须滚动更新 |
| 文件名含 `·` 作为分隔符 | 统一使用 `｜`（全角竖线） |
| 在 `/tmp/` 或 `~/Desktop/` 存文件 | 不走同步管道 |
